 - Keep track of the current confirmed user location in a variable <user-geo-position>
 - Keep track of the tentative possible user location in a variable <maybe-user-geo-position>
 - Keep track of the tentative possible user location's accuracy in meters in a variable <maybe-user-geo-position-accuracy-meters>
 - When client starts app and has geolocation permissions granted already:
    - Set <maybe-user-geo-position> to a newly polled user location
    - Set <maybe-user-geo-position-accuracy-meters> to the accuracy in meters of the newest polled position
    - Set <user-geo-position> to <maybe-user-geo-position>
 - When client starts app and does not have geolocation permissions granted already:
    - On granting geolocation permission:
       - Set <maybe-user-geo-position> to a newly polled user location
       - Set <maybe-user-geo-position-accuracy-meters> to the accuracy in meters of the newest polled position
       - Set <user-geo-position> to <maybe-user-geo-position>
 - Every <specified-amount-of-time>:
    - Update <maybe-user-geo-position> to a newly polled user location
    - Update <maybe-user-geo-position-accuracy-meters> to the accuracy in meters of the newest polled position
    - If <maybe-user-geo-position> with a radius of <maybe-user-geo-position-accuracy-meters> includes <user-geo-position>:
       - Don't update <user-geo-position>
    - Else:
       - Update <user-geo-position> to <maybe-user-geo-position>
 - Whenever user geolocation will be used on the frontend:
    - Use <user-geo-position> instead of directly using what the browser's geolocation returns
